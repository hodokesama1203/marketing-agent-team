## 11. GO / NO-GO 게이트

다음 항목이 모두 체크되어야 데이터 기준 `GO 후보`다. 하나라도 미체크면 `NO-GO`다.

- [ ] 가격·VAT·SKU/옵션·옵션별 재고가 증빙과 함께 확정됨
- [ ] 24mm 측정 정의, 공차, 나사 규격, 호환 조건과 제외 범위가 기술 승인됨
- [ ] 구성품·설치·치수·재질·제조자·원산지가 증빙과 일치함
- [ ] 인증/면제 및 법정 표시 근거가 승인됨
- [ ] 배송 마감·소요일·배송비·반품/교환·A/S가 확정됨
- [ ] 실제 SKU 이미지와 모든 상업 이용 권리가 확인됨
- [ ] 네이버·쿠팡 판매자 계정과 최소권한 책임자가 준비됨
- [ ] 두 플랫폼의 현재 공식 운영 문서 URL·접근일·적용 규칙이 기록됨
- [ ] 모든 필수 검토자가 승인했고 미해결 필수 항목이 없음
- [ ] 최종 등록 payload의 가격·재고·배송 값을 실행 직전에 재검증할 책임자가 지정됨

- 데이터 준비 판정 [필수]: `GO 후보` / `NO-GO`
- 미통과 항목 ID·사유 [필수, 없으면 `없음`]:
- 보완 책임자·기한 [필수, NO-GO 시]:
- 판정자·판정일 [필수]:

> `GO 후보`는 편집·패키지 작성 단계로 넘길 수 있다는 뜻일 뿐이다. 실제 네이버/쿠팡 등록, 가격 입력, 재고 반영, 게시 또는 광고는 정확한 실행 내용에 대한 사용자의 별도 명시적 승인과 적절한 계정 접근이 있어야 한다.

## 12. 다음 핸드오프

1. 작성자는 완성된 본 폼과 모든 증빙을 `marketplace-commerce`에 전달한다.
2. Marketplace Commerce Agent는 추정 없이 네이버·쿠팡별 등록 payload 초안을 작성하고, 증빙표·미해결 항목·공식 운영 문서 기록을 결합한다.
3. Campaign Director는 출처, 상호 일관성, 승인 상태를 검토해 사용자에게 편집/실행 결정을 요청한다.
4. 사용자가 정확한 플랫폼별 payload와 외부 행동 범위를 별도로 승인하기 전에는 등록·가격 설정·게시하지 않는다.
